Store

Dieser Dienst wird zur Speicherung von DICOM Bildern verwendet. Er zählt zu den grundlegenden Eigenschaften des DICOM Netzwerkdienstes und kommt dementspechend auch häufig zum Einsatz. Nach jeder abgeschlossenden Studie eines Patienten werden die Bilder mit dem DIMSE C-Store an ein Archiv versendet. Zur Speicherung in einer Datenbank/Langzeitarchiv. Die Workstations speichern die abgerufenen Bilder auch ab, jedoch werden diese mit einem andern DIMSE Befehl angefordert. Der Store Netzwerkdienst kann als züsätlicher Teil in anderen Serviceklassen enthalten sein.

Im folgenden wird auf der Workstation aufgezeigt, was damit gemeint ist. Dazu wird jeweils dasselbe auf dem RPi gezeigt mit einer Anleitung dazu.

Tutorial

PC

  1. Nach dem öffnen des Orthanc Explorers im Webbrowser wählt man eine Studie aus.
  2. Auf der linken Seite ist ein Bedienfeld mit dem Namen Send to DICOM Modality.
  3. Danach wählt man die gewünschte Modalität aus.

Workstation

Mit dem Sante PACS Server lassen sich ebenfalls DICOM Bilder versenden.

  1. Nach dem man einen Patienten angewählt hat, kann der Balken Send to C-Store angewählt werden.

  2. Es erscheint ein neues Fenster, indem man den C-STORE SCP wählen kann.

  3. Nach dem senden kann mit dem Öffnen des Network Report SCU der Verlauf kontrolliert werden im Network Report.

  4. Dazu macht man einen Doppelklick auf den ausgeführten Service Report und der Network Service Log erscheint.

Hier ist es wie mit dem Raspberry Pi und der debug option möglich, die gesendeten Informationen auszulesen.

RPi

  1. Sobald die Verbindung mit dem RPi steht, kann folgender Befehl in die Konsole eingegeben werden. 
    storescu 192.168.1.20 4444 asdf.dcm
  2. Mit dem verbose mode (-v) lassen sich zusätliche Infos anzeigen 
    storescu 192.168.1.20 4444 asdf.dcm -v
  3. Wenn anstelle von verbose der debug mode (-d) aktiviert wird, erhält man eine ausführliche Liste der Übertragung. 
    storescu 192.168.1.20 4444 asdf.dcm -d
